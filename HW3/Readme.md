# Построение Underlay сети (ISIS + BFD) в архитектуре Leaf-Spine

Этот репозиторий содержит конфигурационные файлы и документацию для построения Underlay-сети с использованием протокола ISIS и BFD в архитектуре Leaf-Spine.

## 1. Топология сети

Архитектура состоит из 2-х Spine коммутаторов и 3-х Leaf коммутаторов.
- К `leaf1` и `leaf2` подключено по одному серверу.
- К `leaf3` подключено два сервера.

## 3. Логика работы ISIS и BFD

### 3.1. ISIS (Intermediate System to Intermediate System)

Протокол ISIS используется для построения Underlay-сети, обеспечивая полную IP-связность между всеми коммутаторами (Leaf и Spine). Это является фундаментом для дальнейшего построения оверлейных сервисов (например, VXLAN).

- **`router isis ISIS_NET`**: Запускает процесс ISIS с именем экземпляра `ISIS_NET`.
- **`net 49.0001.{router-id}.00`**: Устанавливает NET (Network Entity Title) для маршрутизатора. `49.0001` - это `private area address`, а `{router-id}` - это уникальный идентификатор системы, основанный на IP-адресе `Loopback0`.
- **`is-type level-2`**: Все маршрутизаторы работают в режиме `level-2`, что эквивалентно магистральной зоне OSPF.
- **`isis circuit-type level-2`**: Эта команда используется на всех линках между Leaf и Spine для оптимизации работы ISIS.
- **`passive-interface <interface>`**: Эта команда выборочно делает интерфейсы пассивными.
- **`redistribute connected`**: Анонсирует подключенные сети в домен ISIS.

### 3.2. BFD (Bidirectional Forwarding Detection)

BFD используется для сверхбыстрого обнаружения сбоев на пути между двумя маршрутизаторами. В данной конфигурации BFD интегрирован с ISIS для ускорения сходимости сети.

- **`bfd interval 300 min-rx 300 multiplier 3`**: Эта команда настраивается на каждом физическом интерфейсе между Leaf и Spine.
    - `interval 300`: BFD-пакеты отправляются каждые 300 миллисекунд.
    - `min-rx 300`: Ожидаемый минимальный интервал приема BFD-пакетов от соседа - 300 мс.
    - `multiplier 3`: Если 3 последовательных BFD-пакета не будут получены от соседа, сессия BFD будет считаться разорванной.
    - **Итоговое время обнаружения сбоя:** 300 мс * 3 = 900 мс.
- **`bfd all-interfaces`**: Эта команда в секции `router isis ISIS_NET` автоматически включает интеграцию BFD со всеми интерфейсами, на которых активен ISIS.

## 5. Порядок развертывания и проверки

1.  Загрузить соответствующую конфигурацию на каждое устройство.
2.  Применить сетевые настройки на серверах в соответствии с таблицей в п. 2.3.
3.  **Проверка ISIS и интеграции с BFD:**
    - На каждом коммутаторе выполнить команду `show isis neighbors detail`. В выводе для каждого соседа должно быть `BFD-status: Up`. Это подтверждает, что и ISIS, и BFD сессии активны и интегрированы.
    - На любом коммутаторе выполнить `show ip route isis`, чтобы проверить, что в таблице маршрутизации присутствуют маршруты ко всем `Loopback` интерфейсам и `VLAN` интерфейсам других коммутаторов.
4.  **Проверка связности:**
    - С каждого коммутатора выполнить `ping` до `Loopback` IP-адресов всех остальных коммутаторов в сети.
    - С сервера `server1` (`172.16.10.1`) выполнить `ping` до своего шлюза `172.16.10.254`.

## 6. Проверка ECMP (Equal-Cost Multi-Path)

ECMP позволяет распределять трафик по нескольким путям с одинаковой стоимостью. В нашей топологии у каждого Leaf-коммутатора есть два равноценных пути до любой удаленной подсети через `spine1` и `spine2`.

Наличие ECMP можно проверить командой `show ip route` на любом из Leaf-коммутаторов.
