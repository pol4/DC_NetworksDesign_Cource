# Проект сетевой фабрики Leaf-Spine (2 Spines, 3 Leaves)

Этот репозиторий содержит конфигурационные файлы и документацию для построения сетевой фабрики по архитектуре Leaf-Spine.

## 1. Топология сети

Архитектура состоит из 2-х Spine коммутаторов и 3-х Leaf коммутаторов.
- К `leaf1` и `leaf2` подключено по одному серверу.
- К `leaf3` подключено два сервера.

## 2. План IP-адресации

### 2.1. Underlay Network (Инфраструктурная сеть)

- **Loopback-интерфейсы (`/32`):** Блок `192.168.0.0/24`. Используются как стабильные Router-ID и для установки BGP-сессий.
- **Point-to-Point линки (`/31`):** Блок `10.0.0.0/24`. Используются для физического соединения Leaf и Spine коммутаторов.

| Устройство | Интерфейс | IP-адрес/Маска | Назначение |
|------------|-------------|------------------|--------------------------------|
| **Spines** |             |                  |                                |
| spine1     | Loopback0   | `192.168.0.1/32` | Router-ID, BGP-пиринг          |
|            | Ethernet1   | `10.0.0.0/31`    | Линк к leaf1                   |
|            | Ethernet2   | `10.0.0.2/31`    | Линк к leaf2                   |
|            | Ethernet3   | `10.0.0.4/31`    | Линк к leaf3                   |
| spine2     | Loopback0   | `192.168.0.2/32` | Router-ID, BGP-пиринг          |
|            | Ethernet1   | `10.0.0.6/31`    | Линк к leaf1                   |
|            | Ethernet2   | `10.0.0.8/31`    | Линк к leaf2                   |
|            | Ethernet3   | `10.0.0.10/31`   | Линк к leaf3                   |
| **Leaves** |             |                  |                                |
| leaf1      | Loopback0   | `192.168.0.101/32`| Router-ID, BGP-пиринг, VTEP    |
|            | Ethernet1   | `10.0.0.1/31`    | Линк к spine1                  |
|            | Ethernet2   | `10.0.0.7/31`    | Линк к spine2                  |
| leaf2      | Loopback0   | `192.168.0.102/32`| Router-ID, BGP-пиринг, VTEP    |
|            | Ethernet1   | `10.0.0.3/31`    | Линк к spine1                  |
|            | Ethernet2   | `10.0.0.9/31`    | Линк к spine2                  |
| leaf3      | Loopback0   | `192.168.0.103/32`| Router-ID, BGP-пиринг, VTEP    |
|            | Ethernet1   | `10.0.0.5/31`    | Линк к spine1                  |
|            | Ethernet2   | `10.0.0.11/31`   | Линк к spine2                  |

## 3. План BGP

Используется eBGP для обмена маршрутами.
- **Spine-коммутаторы** находятся в одной AS: `65000`.
- **Leaf-коммутаторы** находятся в уникальных AS.

| Устройство | Номер AS (ASN) |
|------------|----------------|
| spine1     | 65000          |
| spine2     | 65000          |
| leaf1      | 65001          |
| leaf2      | 65002          |
| leaf3      | 65003          |

- **Пиринг:** BGP-сессии устанавливаются между Loopback-адресами коммутаторов.
- **Анонсы:**
  - Все коммутаторы анонсируют свои loopback-адреса в BGP.
  - Leaf-коммутаторы дополнительно анонсируют подсети серверов (Overlay).

## 5. Порядок развертывания и проверки

1.  Загрузить соответствующую конфигурацию на каждое устройство.
2.  Применить сетевые настройки на серверах в соответствии с таблицей в п. 2.3.
3.  **Проверка Underlay:**
    - С любого Leaf-коммутатора выполнить `ping` до Loopback-адресов всех Spine-коммутаторов (например, с `leaf1` -> `ping 192.168.0.1`).
    - С любого Spine-коммутатора выполнить `ping` до Loopback-адресов всех Leaf-коммутаторов.
4.  **Проверка BGP:**
    - На каждом устройстве выполнить команду `show ip bgp summary`. Убедиться, что все сессии находятся в состоянии `Established`.
5.  **Проверка Overlay:**
    - На `spine1` или `spine2` выполнить `show ip route`. Убедиться, что в таблице маршрутизации присутствуют маршруты к серверным подсетям (`172.16.10.0/24`, `172.16.20.0/24`, `172.16.30.0/24`) через ECMP (несколько равнозначных путей).
    - С `server1` (`172.16.10.1`) выполнить `ping` до `server2` (`172.16.20.1`). Трафик должен успешно проходить.
